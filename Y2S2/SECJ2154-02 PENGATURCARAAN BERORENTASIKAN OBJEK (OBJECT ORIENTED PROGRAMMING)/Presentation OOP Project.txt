2) Introduction for our project:

Our project is a digital bakery ordering system. It allows users to log in, view bakery items such as cakes, breads, pastries, and drinks, place orders, and complete payments. Admins can view users and orders. This system was built fully using Java and OOP concepts.

3) Problems Solved:

- Manual bakery ordering is inefficient.
- No proper way to track users or previous orders.
- Cash-only payments and no way to manage menus digitally.


4) Key Features:

| Feature               | Description                                         |
| --------------------- | --------------------------------------------------- |
| **Login/Register**    | Authenticated user system, with roles (User/Admin). |
| **Role-Based Menus**  | Admins can view users/orders. Users can shop.       |
| **Cart and Checkout** | Real-time cart with quantity & special requests.    |
| **File Handling**     | Data saved to `users.txt` and `orders.txt`.         |
| **GUI**               | Implemented using `JOptionPane` (Swing-based GUI).  |

5) Java Concept used:

| OOP Concept        | In Your Code                                                       |
| ------------------ | ------------------------------------------------------------------ |
| **Class & Object** | `User`, `Order`, `FoodItem`, etc.                                  |
| **Encapsulation**  | Private fields with `get/set` methods.                             |
| **Inheritance**    | Not applied directly but roles are modeled with enums.             |
| **Polymorphism**   | Overloaded methods (`toString()`, etc).                            |
| **Abstraction**    | Logical separation via helper classes like `AuthenticationSystem`. |
